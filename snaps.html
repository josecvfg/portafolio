Snaps

Notas que viven el tiempo que tú decides. Luego, desaparecen.

Flutter Go Redis iOS & Android
01 — El proyecto

Una nota tiene
fecha de
expiración.

Snaps nació de una idea simple: ¿qué pasaría si tus notas tuvieran vida propia? No todo merece ser guardado para siempre. A veces solo necesitas capturar un pensamiento, una tarea, un recordatorio, y luego dejarlo ir.

Con Snaps, tú decides cuánto tiempo vive cada nota. Puede ser 5 minutos, 2 horas, o 3 días. Cuando el tiempo termina, la nota desaparece sin dejar rastro. Sin archivar, sin eliminar manualmente, sin acumulación.

La idea conecta con algo más profundo: la efimeralidad como característica, no como bug. El olvido, por diseño.

02 — Características
001
Duración configurable

Cada nota tiene su propio timer. Elige desde minutos hasta días. El sistema de expiración corre en el backend sin depender del dispositivo.

002
Personalización visual

Colores, fondos y estilos por nota. Cada snap puede tener su propia identidad visual para que sea fácil de identificar de un vistazo.

003
Widget nativo

Tus notas activas siempre visibles en la pantalla de inicio. El widget muestra el contenido y el tiempo restante en tiempo real.

004
Expiración garantizada

El borrado es real y permanente. Cuando termina el tiempo, el backend elimina la nota de Redis. No hay recuperación posible.

005
Performance nativa

Flutter compila a código nativo en ambas plataformas. Las animaciones corren a 60fps y la app se siente rápida en cualquier dispositivo.

006
Backend de baja latencia

Go maneja las requests con mínimo overhead. Redis gestiona los timers de expiración de manera nativa con TTL por clave.

04 — Arquitectura técnica
📱
Flutter
Cliente
App nativa para iOS y Android. UI construida con widgets personalizados, animaciones fluidas y sincronización en tiempo real con el backend.
⟶
⚙️
Go API
Backend
API REST de alto rendimiento. Gestiona autenticación, creación de snaps y coordina la expiración con Redis. Baja latencia y mínimo consumo de recursos.
⟶
⚡
Redis
Almacenamiento
Almacena las notas con TTL nativo. Cuando una clave expira, Redis la elimina automáticamente. Sin crons, sin jobs externos: la expiración es atómica.
Flutter → Go
Comunicación
La app hace requests HTTP/REST a la API de Go. El token de sesión viaja en headers. Respuestas en JSON, parsing automático con modelos Dart tipados.
Go → Redis
Expiración con TTL
Cada snap se guarda en Redis con un TTL exacto en segundos. Go calcula el TTL al crear la nota. Redis maneja el borrado sin ningún proceso adicional.
Widget
Sincronización nativa
El widget de pantalla de inicio usa las APIs nativas de iOS (WidgetKit) y Android para mostrar el snap activo y actualizar el countdown en background.
