Snaps - backend

DigitalOceanRedisGitHub Actions

El backend de Snaps corre en un Droplet de DigitalOcean. La API en Go y Redis comparten servidor, coordinados con Docker Compose. GitHub Actions maneja el CI/CD: cada push a main compila la app de Flutter, corre los tests y despliega automáticamente.

DGSC - app web

AWS EC2DockerNext.js

DGSC corre en una instancia EC2 de AWS. La app de Next.js está containerizada con Docker, lo que hace el deploy reproducible y el entorno idéntico entre desarrollo y producción. EC2 expone la app directamente vía puerto 443 con certificado SSL.
